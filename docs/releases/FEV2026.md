## Visão Geral

Este documento registra o estado completo do sistema FBTax Cloud ao final de fevereiro de 2026 — o primeiro mês de desenvolvimento. O sistema saiu do zero até uma plataforma completa de apuração tributária com IA, cobrindo Simulador da Reforma Tributária, Apuração Assistida com XMLs de NF-e, Consulta Inteligente por linguagem natural, e integração com a API da Receita Federal.

## Módulos Implementados

### 1. Autenticação (JWT)
- Login / Logout / Registro / Recuperação de senha
- Roles: `admin` / `user`
- Email de confirmação e reset via SMTP
- Rotas protegidas (`ProtectedRoute` + `AdminRoute`)

### 2. Simulador da Reforma Tributária (Fases 1 e 2)
**Entrada de dados:** Upload de SPED EFD ICMS/IPI e EFD Contribuições
**Processamento:** Worker pool (3 workers) com checkpointing por batch
**Registros suportados:** C100, C190, D100, D500, C500

| Página | Rota | Descrição |
|---|---|---|
| Importar EFD | `/importar-efd` | Upload SPED com progresso em tempo real |
| Operações Comerciais | `/mercadorias` | Filtros: Filial, Mês, Movimento (Entrada/Saída), Tipo CFOP |
| Operações Simples Nacional | `/operacoes/simples` | Detalhamento fornecedores Simples Nacional |
| Dashboard | `/dashboards` | Visão geral com gráficos |

**Projeções IBS/CBS:**
- Calculadas dinamicamente baseadas nas alíquotas por ano (2026–2033)
- `aliquota_ibs_uf + aliquota_ibs_mun + aliquota_cbs` da `tabela_aliquotas`
- `vl_ibs_projetado = vl_doc * (ibs_rate / 100)`

### 3. Resumo Executivo com IA
**Rota frontend:** `/relatorios/resumo-executivo`
**API:** POST `/api/ai/report` + GET `/api/ai/report`

- Narrativa gerada pela Z.AI GLM (modelo `glm-4-plus`)
- Gerado automaticamente após importação SPED (worker)
- Botão "Regenerar" para solicitar novo relatório
- Cache em `ai_reports` com `company_id + periodo`

**Email estruturado (v5.9.0):**
Ao regenerar, envia e-mail para todos os gestores da empresa com:
1. KPIs (Faturamento Bruto, Total Entradas, ICMS detalhado)
2. Projeção Reforma 2033 (barra visual HTML + tabela comparativa)
3. Carga Tributária Efetiva (alíquotas efetivas ICMS, IBS, CBS, Total Reforma)
4. Comparativo Período Anterior (quando disponível)
5. Créditos IBS+CBS em Risco (alerta vermelho com valor total + detalhamento)
6. Análise da IA (narrativa como comentário, não como fonte de dados)

> **Regra crítica:** O email usa `TaxComparisonData` (dados estruturados do banco) como fonte de verdade. A narrativa da IA é complementar e não é usada para calcular valores financeiros.

### 4. Consulta Inteligente (Text-to-SQL)
**Rota:** `/relatorios/consulta-inteligente`
**API:** POST `/api/ai/query`
**Arquivos:** `backend/handlers/ai_query.go`, `backend/services/text_to_sql.go`

- Usuário digita pergunta em português → IA gera SQL → backend executa → tabela formatada
- Acesso a todas as views materializadas + tabelas de operações
- Histórico de perguntas bem-sucedidas salvo em `localStorage` (max 10)
- Timeout 90s, max_tokens 2048
- Extração robusta: usa **último** bloco ` ```sql ``` ` do response GLM
- Placeholder `__COMPANY_ID__` substituído via regex (captura variações truncadas)
- Valores DECIMAL formatados como moeda pt-BR; CNPJs mascarados por nome de coluna

**Views disponíveis para a IA:**
- `mv_mercadorias_agregada` (faturamento/compras por tipo CFOP)
- `mv_operacoes_simples` (somente Simples Nacional)
- `mv_compras_fornecedores` (todos fornecedores, CFOP < 5000, com tipo_cfop)
- `operacoes_comerciais` + `participants` (via JOIN import_jobs)

### 6. Apuração Assistida — NF-e

#### NF-e Saídas (mod 55 e 65)
**Importar:** `/apuracao/saida` → POST `/api/nfe-saidas/upload`
**Consultar:** `/apuracao/saida/notas` → GET `/api/nfe-saidas`
**Tabela:** `nfe_saidas`

- Upload de múltiplos XMLs por pasta (arrastar e soltar)
- Filtros: mês/ano + CNPJ cliente
- Dialog de detalhe ao clicar: Identificação, Emitente, Destinatário, ICMSTot, IBSCBSTot
- IBS/CBS nullable (muitos fornecedores antigos não têm a tag)

#### NF-e Entradas (mod 55)
**Importar:** `/apuracao/entrada` → POST `/api/nfe-entradas/upload`
**Consultar:** `/apuracao/entrada/notas` → GET `/api/nfe-entradas`
**Tabela:** `nfe_entradas`

- Rejeita automaticamente XMLs com `tpNF=1` (saídas)
- Filtros: mês/ano + CNPJ fornecedor
- IBS/CBS **DEFAULT 0 NOT NULL** — fornecedores sem a tag ficam com 0 (não null)
- Dialog de detalhe: Identificação, Fornecedor, Destinatário, ICMSTot, IBSCBSTot

**Diferenças críticas Saída vs Entrada:**

| Aspecto | Saída | Entrada |
|---|---|---|
| `tpNF` | `== "1"` | `== "0"` |
| Coluna emitente | `emit_*` | `forn_*` |
| IBS/CBS | nullable | `DEFAULT 0 NOT NULL` |
| Índice extra | `emit_cnpj` | `forn_cnpj` |

### 7. Créditos IBS+CBS em Risco
**Rota:** `/apuracao/creditos-perdidos`
**API:** GET `/api/creditos-perdidos`
**Handler:** `backend/handlers/creditos_perdidos.go`

Duas fontes de crédito perdido:
1. **NF-e sem IBS/CBS:** `SELECT SUM(v_nf) FROM nfe_entradas WHERE v_ibs = 0 AND v_cbs = 0 AND forn_cnpj != dest_cnpj (mesmo grupo)`
2. **Fornecedores Simples Nacional:** `SELECT SUM(total_valor) FROM mv_operacoes_simples WHERE company_id = $1`

Crédito projetado perdido = `(valor_total) × (ibs_rate + cbs_rate) / 100`
Alíquotas padrão: IBS 17,7% + CBS 8,8% = Total 26,5% (se não configurado)

> **Regra anti-duplicação:** Exclui transferências internas (mesmo CNPJ raiz — 8 primeiros dígitos).

### 8. Integração API Receita Federal (RFB)
**Rota:** `/rfb/*`

| Página | Rota | Descrição |
|---|---|---|
| Credenciais | `/rfb/credenciais` | CRUD OAuth2 (client_id, client_secret, cert) |
| Apuração CBS | `/rfb/apuracao` | Disparar apuração + acompanhar status |
| Gestão Créditos | `/rfb/gestao-creditos` | Visualizar créditos IBS/CBS da RFB |

**Portal:** `https://consumo.tributos.gov.br/`

## Decisões Técnicas Relevantes

### Email HTML Table-based
Email clients (Outlook, Gmail) não renderizam SVG nem Flexbox. Todo HTML de e-mail usa apenas `<table>`, `<tr>`, `<td>` com estilos inline e atributos HTML (`bgcolor`, `width`, `height`).

> A função `generateTaxComparisonSVG()` foi substituída por tabela HTML com barra de progresso usando `bgcolor` e `width%`.

### IBS/CBS em NF-e Entradas: DEFAULT 0
Fornecedores emitindo NF-e antes da Reforma não incluem tags IBS/CBS. Na tabela `nfe_entradas`, todas as colunas IBS/CBS são `DEFAULT 0 NOT NULL`. O handler usa `toDecimal()` em vez de `toNullDecimal()`. Isso garante que o cálculo de créditos em risco funcione via `WHERE v_ibs = 0 AND v_cbs = 0`.

### Worker Pool
3 workers paralelos processam jobs de importação SPED. O refresh das views materializadas ocorre apenas no último job (`pendingCount == 0`) para evitar refreshes desnecessários durante importações em lote.

### Docker Layer Cache (GHA)
O cache `type=gha` do GitHub Actions pode servir camadas stale. Para forçar rebuild, modificar qualquer arquivo dentro de `COPY backend/ ./` (ex: comentário de versão em `main.go`).

### Placeholder Text-to-SQL
A IA GLM às vezes trunca o placeholder `__COMPANY_ID__` para `__COMPANY` ou `__COMPANY_ID`. A substituição usa regex `'__COMPANY[^']*'` para capturar todas as variações antes de executar o SQL no banco.
